1.동기 비동기 개념 동기(Synchronous) - 요청에 대한 응답 완료시 다음 업무를 요청하는 방식, 현재 실행 중인 코드 완료 후 다음 코드를 실행 (장점) 직관적이며 설계가 쉽고 실행순서가 보장된다 (단점) 실행 중인 코드의 처리시간이 늘어나도 결과가 나올 때 까지 대기해야 하는 작업중단(Blocking)현상 발생 비동기(Asynchronous) - 요청에 대한 응답 확인 전 다음 업무도 요청하는 방식, 현재 실행 중인 코드가 완료되지 않은 상태에서 다음 코드를 실행 (장점) 데이터 처리 중 다른 작업이 가능하기 때문에 자원을 효율적으로 사용하며 불필요한 로딩을 줄일 수 있다. (단점) 설계가 어렵고 실행순서가 보장되지 않으므로 어떤 작업이 먼저 끝날지 알 수 없다. 자바스크립트는 기본적으로 인터프리터 방식(한줄씩 해석하고 표현)이자, 동기방식 언어(한번에 한 작업만 수행 가능한 싱글 스레드) 하지만 동기 처리시 Blocking현상이 발생되므로 작업적 효율을 위해 비동기방식(병렬 작업)이 대세가 됨 *Blocking: 해당 작업의 완료까지 화면이 멈추는 상황 비동기 주요 사례 - 마우스, 키보드 입력(click, keydown 등) - 페이지 로딩 (DOMContentLoaded 등) - 타이머 API (setTimeout 등) - 애니메이션 API (requestAnimationFrame) - AJAX - XHR객체를 사용한 비동기 통신 방식 - fetch API - Promise(=비동기처리 결과 반환 객체)기반 비동기 통신 방식
